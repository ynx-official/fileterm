# Desktop UI Regression Checklist

本文记录 TermDock 桌面壳、工作区布局和视觉状态的最小回归清单。后续只要改顶部标签栏、首页、系统侧栏、终端底部控件、文件面板或文件编辑器，都需要按本清单做人工检查。

## 1. 顶部标签栏与窗口工具

- macOS 下红黄绿窗口按钮不能被分隔线、标签页或品牌文字覆盖。
- 首页侧栏收起时，标签页应向左贴近红黄绿按钮后的可用区域。
- 首页侧栏收起时，左侧窄栏顶部需要有水平封口线，竖向分隔线从封口线下方开始。
- 普通标签、系统信息标签和新建标签切换时，内容区应有一致的方向动画。
- 开启 `prefers-reduced-motion` 后，标签切换动画应关闭。
- 工作区焦点按钮应能在“放大/缩小”图标间切换；进入焦点模式时收起系统侧栏和底部文件面板，再次点击恢复。

## 2. 首页与侧栏

- 首页展开态保留 TermDock 品牌、版本、导航、Docs、GitHub。
- 首页收起态只显示可点击图标，不显示挤压或竖排文字。
- 首页深色/浅色主题下，主内容横线、侧栏边界和底部状态栏都要可见。
- 收起态下侧栏图标应保持固定尺寸，不因文字或 hover 状态改变宽度。

## 3. 系统侧栏与监控摘要

- 系统侧栏展开态显示 IP、连接地址、系统信息、CPU、内存、交换、进程、网络和文件系统信息。
- 系统侧栏收起态显示三条细竖向监控柱：CPU、内存、交换。
- 收起态监控柱内部不放文字；名称和百分比显示在柱体下方。
- 收起态监控柱在浅色/深色主题下都应有足够对比度，且不会压到终端内容。
- 收起/展开系统侧栏时，工作区内容不应出现横向跳动或文字竖排。

## 4. SSH 工作区

- 终端输出区域可以延伸到 shell 底部，不为命令输入条预留固定空白。
- 命令输入条悬浮在终端上方，具备半透明雾透效果。
- 命令输入条、历史、选项、闪电发送、复制、搜索、展开菜单按钮都必须可点击。
- 文件面板抽屉按钮与命令输入条在同一水平线上。
- 文件面板展开、收起、恢复上次高度、拖拽高度后再收起恢复都应正常。
- 文件面板收起后，终端区域自动铺满；展开后，本地/远程文件列表、传输状态栏不丢失。

## 5. 文件编辑器

- 文件编辑器窗口应呈现左侧文件树、右侧 Monaco 编辑区。
- 左侧当前文件节点可点击并聚焦编辑器。
- 文件、编辑、搜索、偏好设置、编码、语言等菜单仍可打开。
- `Cmd/Ctrl+S` 保存，`Cmd/Ctrl+F` 触发 Monaco 搜索，不触发终端搜索。
- 深色/浅色主题切换后，Monaco 背景、文字、行号、选区和光标颜色与 TermDock 主题一致。

## 6. 平台资源

- macOS 菜单栏使用 `trayTemplate.png` / `trayTemplate@2x.png` template 图标，图形完整且深浅菜单栏都可见。
- Windows 继续使用 `icon.ico` 作为应用图标，不复用 macOS template 图。
- 打包前确认 `apps/desktop/build/` 下平台资源仍在 electron-builder `files` 范围内。

## 7. 最小验证命令

```bash
npm run typecheck -w @termdock/desktop
npm run build -w @termdock/desktop
```

如果只改文档或纯 CSS，可至少运行：

```bash
git diff --check
```
