# RustSec 通告例外

更新时间：2026-07-17
维护人：FileTerm maintainers
下次强制复查：2026-08-17

## 策略

Tauri 的 `Cargo.lock` 在 CI 中使用 RustSec `audit-check` 扫描。默认情况下，新发现的
漏洞会阻断 CI；只有本文件列出的通告可以被精确忽略。每条例外必须同时具备依赖链、
可达性判断、临时处置和移除条件，不能使用版本范围或全局关闭扫描。

| `RUSTSEC-2026-0194` | `rfd -> wayland-client -> wayland-scanner -> quick-xml 0.39.4`。仅在 Linux Wayland 代码生成阶段解析依赖包内置协议 XML；FileTerm 不向该解析器传入用户或远端 XML，因此产品运行时不可达。 | 保留 Wayland 原生文件对话框支持并精确忽略该 ID；每月检查 `wayland-scanner` / `rfd` 新版本。             | 上游发布使用 `quick-xml >= 0.41.0` 的兼容版本后立即升级并删除例外。      |

| `RUSTSEC-2023-0071` | `russh -> rsa 0.10.0-rc.18`。仅使用 RSA 私钥认证时条件可达；密码、Ed25519 和 ECDSA 认证不经过该 RSA 私钥运算。上游当前没有修复版本。                                                   | 保留既有 RSA 连接兼容性；新建密钥优先使用 Ed25519/ECDSA。该项是已接受的中危剩余风险，不标记为“不可达”。 | `russh` 切换到常量时间 RSA 实现，或产品明确停止 RSA 私钥认证后删除例外。 |

## 复查命令

```bash
cd apps/tauri/src-tauri
cargo audit
cargo tree --target all -i quick-xml@0.39.4
cargo tree --target all -i rsa@0.10.0-rc.18
```

复查时还要确认 `.github/workflows/ci.yml` 与
`apps/tauri/src-tauri/.cargo/audit.toml` 的 ID 完全一致；任何新的漏洞 ID 都必须先失败，
不得顺手加入 ignore 列表。
